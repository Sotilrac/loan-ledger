# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Targets are versioned independently:

- `@loan-ledger/web` via `web-vX.Y.Z` tags
- `@loan-ledger/nextcloud` via `nc-vX.Y.Z` tags

## [Unreleased]

### Added

- Monorepo scaffold: `@loan-ledger/core` and `@loan-ledger/web` workspace packages.
- CI pipeline: lint, typecheck, test, build on every branch push (GitLab CI).
- Lefthook pre-commit/pre-push hooks: prettier, eslint, stylelint, editorconfig-checker, typecheck, vitest.
- `@loan-ledger/core` Phase 1: data model types, JSON Schemas (`loan.schema.json`, `mappings.schema.json`), `parseLoanYaml` / `serializeLoanYaml`, `parseMappingsYaml` / `serializeMappingsYaml`, `validateLoan` / `validateMappings` (ajv), `ingestCsv` (papaparse) with bank-CSV column mapping.
- `@loan-ledger/core` Phase 2: `computeLoan` engine — pure function for scheduled amortization, rate-schedule / ARM re-amortization, actual-vs-scheduled reconciliation, derived summary (equity, projected payoff, months ahead). UTC-midnight date utilities. `buildDemoLoan` / `DEMO_LOAN` fallback data.
- `@loan-ledger/web`: renders demo loan with editorial design tokens (`:root { --ll-* }`) — hero equity number, summary blocks, loan-terms grid, recent-payments table with Marigold actual-row borders.
- `@loan-ledger/core` Phase 3: ScenarioEngine with 10 mutations (recurring_extra_principal, one_time_extra, change_rate, change_term, stop_extra_payments, change_escrow, remove_payment, remove_payments_where, replace_payment, set_original_rate). `evaluateScenario` / `evaluateAllScenarios` producing signed `interest_saved` / `months_sooner` deltas.
- `@loan-ledger/web` visualisations: hand-rolled SVG charts faithful to the style guide — `BalanceChart` (scheduled thin Ink Blue @ 0.7, actual thick Marigold, projected dashed, scenario overlays in muted ink), `CompositionChart` (stacked principal / interest @ 0.5 / escrow / Marigold extra), `EquityGauge` (3/4 arc, Marigold fill). Shared `palette.ts` is the single source of truth for the graphical language.
- `@loan-ledger/web` Phase 4: Pinia store (`stores/loan.ts`), file handling (`useFileHandle` composable with FSA API + `<input type=file>` fallback, IndexedDB handle persistence via `idb-keyval`). FilePicker component, AmortizationTable with selection/highlight linkage to BalanceChart via shared `selectedPeriod`, scroll-to-today on mount. Global "Edit loan" button toggles `LoanEditForm` (editable property, valuation, loan-term fields); derived values update live as you type. Save writes to the FSA handle when available, else triggers a download. Dirty-state indicator in the header.
- `@loan-ledger/web` Phase 5: rate-schedule editor (add / edit / remove / auto-sort-on-change) inside the edit form. CompositionChart now spans the full loan lifetime with a "today" rule and a rate-change marker at each scheduled rate flip; past bars at full opacity, future bars at 60%; bar gap adapts to slot width so 360-month builds actually render. Scenario card is a real button that toggles the chart overlay. Vite `base: './'` so the bundle serves from any subpath. GitLab Pages deploy job publishes `packages/web/dist` on every push to `main`. Equity gauge hero number rendered in Ink Blue per style guide.
- `@loan-ledger/web` Phase 6: CSV import dialog (native `<dialog>`) with header auto-detection, ad-hoc column mapping, amount-sign + date-format pickers, filter-by-column, live preview of up to 5 parsed Payment rows, and optional "save this mapping for next time" writing to `localStorage` under `@loan-ledger/web:mappings` via a new Pinia `mappings` store. Valuation refresh button fetches the `custom_url` source URL, accepts bare number or `{ amount }` JSON, updates `amount` + `as_of`. Always-available **Download YAML** button (save-as). Layout reworked: summary row holds gauge / facts / scenarios side-by-side, charts and table full-width below.
- `@loan-ledger/core`: `MappingsSource` interface alongside `LoanSource`, exported as the seam between the store and per-target mappings storage. The web app keeps `localStorage`; the Nextcloud app uses a shared `.mappings.yaml` so CSV column mappings travel with shared loan files.
- `@loan-ledger/web`: `LocalStorageMappingsSource` implementation; mappings store now reads/writes through it. Legacy JSON-array values are migrated to `MappingsFile` YAML on first read, no user action required.
- `@loan-ledger/nextcloud` Phase 7 scaffold: PHP skeleton (`Application`, `ConfigService`, `FileScanner`, `FileWriter`, `MappingsRepository`), OCS controllers (`LoanController`, `MappingController`, `SettingsController`) wired with `#[ApiRoute]` attributes, optimistic-concurrency `If-Match` writes via Nextcloud `mtime`, per-user folder pref via `IUserConfig` (default `/Ledgers/`). PHPUnit suite (19 tests / 28 assertions) using out-of-tree `nextcloud/ocp` stubs with minimal local stubs for private `OC\` symbols.
- `@loan-ledger/nextcloud` Phase 7 frontend: Vue 3 + Vite SPA, hash-routed (`LoanList`, `LoanDetail`, `Settings`), pinia stores, OCS-backed `LoanSource` + `MappingsSource`. `PageController` serves the SPA at `/apps/loanledger/`, Vite emits to `js/loanledger-main.js` + `css/loanledger-main.css` matching `Util::addScript` / `Util::addStyle` conventions. Rich loan-detail UI lands in M4 (UI extraction); M3 renders the YAML so the OCS pipe is observable end-to-end.
- `Makefile`: `nc-install`, `nc-build`, `nc-test`, `nc-lint`, `nc-format`, `nc-clean`, `nc-shell` targets. `nc-shell` opens a PHP 8.2 container with `packages/nextcloud/` mounted, so PHP development works without a local PHP install.
- GitLab CI: `ci-nextcloud-php-test` and `ci-nextcloud-php-lint` jobs running on `php:8.2-cli`, gated by `changes` rules so JS-only commits don't pay the PHP install cost.
- `@loan-ledger/ui` Phase 8 extraction: shared Vue components (`AmortizationTable`, `BalanceChart`, `EquityGauge`, `LoanEditForm`, `ScenariosPanel`, `CsvImportDialog`, `PaymentEditorRow`, `ScenarioEditor`), shared Pinia stores (`useLoanStore`, `useMappingsStore`), shared format utilities, and shared design tokens / typography. Both targets consume identical scoped CSS so a fix in one applies to both. Element-level base styles (`html`, `body`, `button`, `*`) live in a separate `base.css` that only the standalone imports, so they don't repaint Nextcloud's chrome.
- `@loan-ledger/nextcloud` Phase 8: full feature parity with the standalone in NC. Single-page shell with a Load dropdown (basename only when one folder is configured, `<folder>/<basename>` when multiple), New-loan flow via `OC.dialogs.prompt` and `OC.dialogs.filepicker`, inline Settings dropdown, Edit / Import payments / Save (only while editing). Last-loaded loan persisted to `localStorage` and auto-selected on boot. Onboarding view leads with the user-facing benefit ("see where every payment really goes... run side-by-side scenarios...") and a "Create" button when the configured folder is missing.
- Multi-folder ledgers: `ConfigService` stores `folders: string[]` (with legacy `folder` migration), `FileScanner` walks every configured folder and reports `getMissingFolders` so the frontend can drive the onboarding state, `FolderService` + `POST /api/v1/folders` create folders on demand via `IRootFolder`, `POST /api/v1/loans` accepts an optional `folder` param so multi-folder users pick the destination per loan via the NC filepicker. `MappingsRepository` and the default `createLoan` target the user's primary (head-of-list) folder; the shared `.mappings.yaml` lives there. PHPUnit suite at 32 tests / 47 assertions, including coverage for `FolderService.create` and `getMissingFolders`.
- Theme follows Nextcloud: `body[data-theme-dark|dark-highcontrast]` triggers our dark palette, `body[data-theme-light|light-highcontrast]` forces light, and `body[data-theme-default]` falls through to the system's `prefers-color-scheme` so NC's auto-mode stays in sync with its own chrome.
- Nextcloud-shell normalizers in `packages/nextcloud/src/style/app.css`: scoped `.ll-shell` overrides for NC's global resets that fight our underline-style form fields (`width: 130px`, `min-height: var(--default-clickable-area)`, hover/focus/active 2px halo), bare `dialog { display: block; margin: 0 }` (which kept modal dialogs centered), and the `dl/dt/dd { padding: 12px }` rule that made the chart tooltip rows feel padded.
